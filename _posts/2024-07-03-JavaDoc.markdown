# java.base
## java.io
데이터 스트림과 직렬화, 파일 시스템을 통한 시스템의 입출력을 담당합니다.  
달리 명시된 경우를 제외하고는 기본적으로 클래스나 인터페이스의 생성자나 매소드로 `null`을 넘기면 `NullPointerException`이 발생합니다.  

### BufferedInputSystem
> public class BufferedInputSystem extends FilterInputStream

`BufferedInputStream`은 입력 스트림에서 버퍼를 제공해 줍니다. `BufferedInputStream`은 내부 버퍼 역할을 할 배열을 생성됩니다. 스트림의 바이트가 읽히면 입력 스트림에서 버퍼로 바이트가 넘어와 버퍼를 다시 채웁니다.

> ##### 버전
> 자바 1.0 버전부터 지원합니다.
{: .block-warning }

#### 필드
##### `buf`
> protected volatile byte[] buf

데이터가 저장될 버퍼 배열입니다. 필요에 따라 다른 크기를 가진 배열로 바꿀 수 있습니다.
##### `count`z
> protected int count

버퍼의 크기입니다.
##### `pos`
> protected int pos

다음으로 읽을 바이트의 인덱스를 가리킵니다. 
##### `markpos`
> protected int markpos

마지막으로 `mark` 매소드가 호출되었을 때의 `pos`값을 저장합니다. 만일 `mark` 메소드가 호출된 적이 없다면 `-1`을 반환합니다.
##### `marklimit`
> protected int marklimit

`reset` 메소드에 대한 호출이 실패하기 전에 `mark` 메소드가 호출한 후 앞쪽으로 읽기가 허용되는 최대치 값입니다.

#### 생성자
> public BufferedInputStream(InputStream in)

`BufferedInputStream`을 생성해 추후 사용될 입력 스트림 `in`을 저장합니다. 생성된 내부 버퍼 배열은 `buf`필드에 저장됩니다.

> public BufferedInputStream(InputStream in, int size)

`size` 크기의 `BufferedInputStream`을 생성합니다. 기타 내용은 이상과 같습니다.
#### 메소드
##### `read`
> public int read() throws IOException

데이터의 다음 바이트를 반환합니다. 스트림에 끝에 도달하면 `-1`을 반환합니다.
> public int read(byte[] b, int off, int len) throws IOException

바이트 배열 버퍼 `b`를 `off` 인덱스부터 `len`개 만큼 읽습니다.  
편의상, 스트림의 `read`메소드를 반복하여 가능한 많은 바이트를 읽으려고 시도합니다. 그러다가 다음 조건 중 하나라도 만족하면 멈춥니다.
- `len`만큼 읽었을 때.
- 스트림의 `read` 메소드가 `-1`을 리턴할 때(파일의 끝).
- 스트림의 `available` 메소드가 `0`을 리턴할 때(추가적인 입력 요청이 막혔을 때).

이 메소드는 실제로 읽은 바이트의 수를 반환합니다. 단, 파일의 끝까지 읽었을 경우 `-1`을 반환합니다.

##### `skip`
> public long skip(long n) theows IOException

`n` 바이트 만큼 건너뜁니다. 실제로 건너 뛴 만큼의 바이트 수를 반환합니다.

##### `available`
> public int available() throws IOException

입력 스트림에서 다른 메소드로 인해 막히지 않고 읽을 수 있는 바이트의 수를 어림잡아 반환합니다.

##### `mark`
> public void mark(int readlimit)

입력 스트림에서 현재 위치를 표시합니다. 이 메소드가 호출되고나서 `reset` 메소드가 불려지면 마지막으로 표시된 지점으로 옮기게 됩니다. 이를 통해 `read` 메소드가 같은 바이트를 다시 읽을 수 있게 합니다.  
파라미터로 주어지는 `readlimit`은 표시된 위치가 인식 불가능해지기 전에 읽을 수 있는 바이트 최대값입니다.

##### `reset`
> public void reset() throws IOException

마지막으로 `mark` 매소드가 불린 자리로 입력 스트림의 위치를 바꿉니다. `markpos`가 -1이라면(`mark`메소드가 불려진 적이 없거나 인식 불가능한 위치로 옮겨졌다면) `IOException` 에러가 발생합니다.

##### `markSupported`
> public boolean markSupported()

입력 스트림이 `mark`와 `reset` 메소드를 지원하는지 확인합니다. `BufferedInputStream`이라면 `true`를 반환합니다.

##### `close`
> public void close() throws IOException

입력 스트림을 닫고 이와 관련된 모든 시스템 자원을 헤제합니다. `close()` 메소드가 불려진 이후에는 `read()`, `available()`, `reset()`, `skip()` 메소드가 `IOException` 에러를 발생시킵니다. 이미 닫힌 입력 스트림을 다시 닫으면 아무일도 일어나지 않습니다.

### BufferedOutputStream
> public class BufferedOutputStream extends FilterOutputStream

`BufferedOutputStream`은 버퍼가 있는 출력 스트림을 구현합니다. 이러한 출력 스트림을 설정하면 매 바이트 쓰기마다 기저에 깔린 시스템에 호출하지 않고도 어플리케이션이 출력 스트림에 바이트를 적을 수 있습니다. 

> ##### 버전
> 자바 1.0 버전부터 지원합니다.
{: .block-warning }

#### 필드
##### `buf`
> protected byte[] buf

데이터가 저장될 내부 버퍼입니다.

##### `count`
> protected int count

버퍼 안에서 유효한 바이트 데이터의 개수입니다.

#### 생성자
> public BufferedOutputStream(OutputStream out)

사용하는 출력 스트림에 데이터를 쓰기 위해 버퍼가 있는 새 출력 스트림을 생성합니다.

> public BufferedOutputStream(OutputStream out, int size)

`size` 크기의 `BufferedOutputStream`을 생성합니다. 기타 내용은 이상과 같습니다.

#### 매소드
##### `write`
> public void write(int b) throws IOException

버퍼가 있는 출력 스트림에 바이트를 씁니다. 매개변수가 int인 이유는 쓸 값을 ASCII 코드 값으로 받기 때문입니다.

> public void write(byte[] b, int off, int len) throws IOException

바이트 배열 `b`를 `off` 인덱스부터 `len`개 만큼 씁니다.

##### `flush`
> public void flush() throws IOException

버퍼가 있는 출력 스트림을 비웁니다.

### BufferedReader
> public class BufferedReader extends Reader

입력 스트림에서 텍스트를 읽습니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

#### 생성자
> public BufferedReader(Reader in, int sz)

`sz`크기의 `in`을 사용하는 입력 버퍼를 생성합니다.

> public BufferedReader(Reader in)

기본값인 크기를 갖는 `in` 을 사용하는 입력 버퍼를 생성합니다.

#### 매소드
##### `read`
> public int read() throws IOException

글자 하나를 읽습니다. 읽은 글자를 0 ~ 65535 사이의 정수로 반환합니다. 스트림의 끝에 도달하면 -1을 반환합니다.

> public int read(char[] cbuf, int off, int len) throws IOException

`cbuf` 배열에 담긴 글자를 `off`인덱스부터 `len` 길이만큼 읽습니다.  

편의상, 스트림의 `read`메소드를 반복하여 가능한 많은 바이트를 읽으려고 시도합니다. 그러다가 다음 조건 중 하나라도 만족하면 멈춥니다.
- `len`만큼 읽었을 때.
- 스트림의 `read` 메소드가 `-1`을 리턴할 때(파일의 끝).
- 스트림의 `available` 메소드가 `0`을 리턴할 때(추가적인 입력 요청이 막혔을 때).  

이 메소드는 실제로 읽은 바이트의 수를 반환합니다. 단, 파일의 끝까지 읽었을 경우 `-1`을 반환합니다.

##### `readLine`
> public String readLine() throws IOException

한 줄의 텍스트를 읽습니다. 한 줄이란 `\n`, `\r`, 혹은 `\n\r`, 또는 파일의 끝에 도달함으로서 구분됩니다.

##### `ready`
> public boolean ready() throws IOException

해당 스트림이 읽힐 준비가 되었는지 말해줍니다. 버퍼가 비어있지 않거나 스트림이 준비가 되었으면 읽힐 준비가 된 것입니다.

##### `markSupported`
> public boolean markSupported()

해당 스트림이 `mark()` 메소드를 지원하는지 말해줍니다.

##### `mark`
> public void mark(int readAheadLimit) throws IOException

스트림의 현재 위치를 표시합니다. 이어지는 `reset()` 메소드 호출은 스트림을 표시된 위치로 옮기려 시도할 것입니다.  
`readAheadLimit`을 지정해 `reset` 메서드가 이 값 이상 읽지 읽으려 하면 작동하지 않게 합니다.

##### `reset`
> public void reset() throws IOException

가장 최근에 표시된 위치로 스트림을 위치시ㅣㅂ니다.

##### `close`
> public void close() throws IOException

스트림을 닫고 이와 관련된 모든 시스템 자원을 헤제합니다. `close()` 메소드가 불려진 이후에는 `read()`, `ready()`, `mark()`, `reset`, `skip()` 메소드가 `IOException` 에러를 발생시킵니다. 이미 닫힌 스트림을 다시 닫으면 아무일도 일어나지 않습니다.

##### `lines`
> public Steram<String> lines()

해당 BufferedReader에서 읽어진 줄인 스트림을 반환합니다. 스트림은 지연 연산됩니다, 즉 터미널 스트림 동작(terminal stream operation)이 진행되는 동안 읽기가 발생할 수 있습니다. 이 때 읽는 주체가 작동하면 터미널 스트림 동작의 결과는 `undefined`가 될 것입니다. 터미널 스트림 동작이 실행된 이후에는 읽는 주체가 다음 글자나 줄을 읽을 특정한 위치에 있을 것이라고 보장할 수 없습니다.  

`BufferedReader`에 접근할 때 `IOException`이 발생한다면 이 메소드는 `UncheckedIOException`에 래핑됩니다. `BufferedReader`가 이미 닫혔다면 이 메소드는 스트림을 반환할 것이며, `BufferedReader`에 작동하는 어떤 동작도 `UncheckedIOExcption`을 일으킬 것입니다.

> ##### 버전
> 자바 1.8 버전부터 지원합니다.
{: .block-warning }

### BufferedWriter
> public class BUfferedWrited extends Writer

출력 스트림에 텍스트를 씁니다. 효율적으로 글자, 배열, 스트링이 적힐 수 있도록 하기 위해 버퍼를 사용합니다. 따로 지정해주지 않는 한 버퍼의 사이즈는 기본값으로 지정됩니다. 이 사이즈는 충분히 크기 때문에 대부분의 작업에 적합할 것입니다.  
모든 플랫폼이 `\n`을 줄의 끝에 쓰는 것은 아닙니다. 하지만 `newLine()` 매소드는 시스템 속성인 `line.separator`에 정의된 바를 줄 구분자로 사용합니다. 그렇기에 이 메소드는 새 줄을 나타내는 줄 구분 문자를 적을 때 특히나 유용합니다.  
일반적으로 쓰는 주체는 즉시 글자나 바이트 스트림으로 출력 값을 전송합니다. 프롬프트 출력이 요구되지 않는 한 `BufferedWriter`을 `FileWriters` 나 `OutputStreamWriters`와 같이 `write()` 동작이 비용이 많이 드는 쓰는 주체에 래핑하는 것이 좋을 것입니다. 다음은 예시입니다.
```Java
PrintWriter out = new PrintWriter(new BufferedWriter(new FileWriter("foo.out")));
```
이 예시에서 `BufferedWriter`은 `PrintWriter`의 출력을 파일에 버퍼를 만듭니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

#### 생성자
> public BufferedWriter(Writer out)

기본 사이즈의 출력 버퍼를 사용하는 글자 출력 스트림을 만듭니다.

> public BufferedWriter(Writer out, int sz)

`sz` 사이즈의 출력 버퍼를 사용하는 글자 출력 스트림을 만듭니다.

#### 매소드
##### `write`
> public void write(int c) throws IOException

글자 하나를 씁니다. 매개변수가 int인 이유는 쓸 값을 ASCII 코드 값으로 받기 때문입니다.

> public void write(char[] cbuf, int off, int len) throws IOException

`char` 배열 `cbuf`를 `off` 인덱스부터 `len`개 만큼 씁니다.

> public void write(String s, int off, int len) throws IOException

String `s`를 `off` 인덱스부터 `len`개 만큼 씁니다. `len`이 음수이거나 `off` + `len`이 음수이면 아무것도 쓰지 않습니다.

##### `newLine`

시스템 속성 `line.separator`에 정의된 줄 구분자를 씁니다.

##### `flush`
> public void flush() throws IOexception

스트림을 비웁니다.

##### `close`
> public void close() throws IOException

스트림을 닫습니다, 이 때 가장 먼저 스트림을 비웁니다. 스트림이 닫힌 이후에 `write()`이나 `flush()` 메소드는 `IOException`을 일으킬 것입니다. 이미 닫힌 스트림을 다시 닫으면 아무 일도 일어나지 않습니다.

### ByteArrayInputStream
> public class ByteArrayInputStream extends InputStream

`ByteArrayInputStream`은 내부 버퍼를 가지고 있어 스트림에서 읽어 들어오는 바이트를 수용할 수 있습니다. 내부 카운터가 있어 `read`메소드로 전달되는 다음 바이트의 위치를 기록할 수 있습니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

#### 필드
##### `buf`
> protected byte[] buf

스트림에서 읽혀지는 바이트의 배열입니다.

##### `pos`
> protected int pos

다음으로 읽혀질 바이트의 인덱스입니다.

##### `mark`
> protected int mark

스트림에서 표시 된 지점입니다. `ByteArrayInputStream`은 기본값으로 위치 0에 표시가 되어있고 `mark()`메소드로 다른 지점에 표시됩니다. 버퍼 위치는 `reset()` 메소드로 이 지점으로 옮겨질 수 있습니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

##### `count`
> protected int count

버퍼의 인덱스입니다.

#### 생성자
> public ByteArrayInputStrem(byte[] buf)

`ByteArrayInputStream`을 생성해 `buf` 배열을 버퍼로 사용합니다.

> public ByteArrayInputStream(byte[] buf, int offset, int length)

`ByteArrayInputStream`을 생성해 `buf` 배열을 버퍼로 사용합니다. `offset`은 `pos`의 초기값이 되고, `offset` + `length`와 `buf` 중 작은 값이 `count`의 초기값이 됩니다.

#### 매소드
##### `read`
> public int read()

입력 스트림에서 데이터의 다음 바이트를 읽습니다. 0 ~ 255 사이의 정수 값을 반환합니다. 단, 더 이상 읽을 바이트가 없으면 -1을 반환합니다.  
read 매소드는 다른 메소드가 실행되는 것을 막을 수 없습니다.

> public int read(byte[] b, int off, int len)

입력 스트림에서 `len` 만큼의 바이트를 읽습니다. `pos`가 `count`와 같다면 파일의 끝이라는 뜻으로 -1을 반환합니다. 그렇지 않다면, 바이트를 읽은 만큼인 숫자 k를 두어 `len`과 `count` - `pos` 중 작은 수와 값이 같게 합니다. 만일 k가 양수라면, 바이트 버퍼 `buf[pos]`부터 `buf[pos + k - 1]`이 `b[off]`부터 `b[off + k - 1]`로 복사됩니다. 이는 `System.arraycopy`와 같은 방식으로 이루어집니다. k값이 pos에 더해진 후 k가 반환됩니다.  
read 메소드는 다른 메소드가 실행되는 것을 막을 수 없습니다.

##### `readAllBytes`
> public byte[] readAllBytes()

입력 스트림에서 남은 모든 바이트를 읽습니다. 이 메소드는 모든 바이트를 읽어 파일에 끝에 도달하거나 혹은 예외가 발생할 때 까지 다른 메소드의 실행을 막습니다. 이 메소드는 입력 스트림을 닫지 않습니다.  
스트림이 끝에 닫은 후 이 메소드를 실행하면 빈 바이트 배열을 반환할 것입니다.  
이 메소드는 많은 양의 데이터를 읽어들이는데 부적합합니다. 모든 바이트를 바이트 배열에 넣어 읽는 것이 간편한 간단한 경우에만 사용되도록 의도되었습니다.  
I/O 에러가 발생한다면 즉시 이 스트림을 닫는 것을 권장합니다.

##### `readNBytes`
> public int readNBytes(byte[] b, int off, int len)

주어진 바이트 배열에 요청된 수의 바이트를 읽습니다. 이 메소드는 `len` 바이트의 입력 데이터를 읽거나 스트림의 끝에 닫거나 예외가 발생할때 까지 다른 메소드의 실행을 막습니다. 실제로 읽은 바이트의 수가 반환됩니다. 아 메소드는 입력 스트림을 닫지 않습니다.  
I/O 에러가 발생한다면 즉시 이 스트림을 닫는 것을 권장합니다.

##### `skip`
> public long skip(long n)

입력 스트림에서 n 바이트만큼 건너 뜁니다. 실제로 건너 뛴 만큼인 숫자 k를 두어 `n`과 `count` - `pos` 중 작은 값과 같게 됩니다. 값 k는 pos에 더해진 뒤에 반환됩니다.

##### `available`
> public int available()

입력 스트림에서 읽히거나 건너 뛰어질 수 있는 만큼인 남은 바이트의 수를 반환합니다.

##### `markSupported`
> public boolean markSupported()

해당 입력 스트링이 mark/reset 메소드를 지원하는지 확인합니다. `ByteArrayInputStream`은 해당 메소드를 지원함으로 반드시 `true`만을 반환핣니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

##### `mark`
> public void mark(int readAheadLimit)

이 스트림에서 현재 표시된 지점을 설정합니다.  
이 클래스에서 `readAheadLimit`은 이무런 의미가 없습니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

##### `reset`
> public void reset()

버퍼를 표시된 지점으로 원위치시킵니다.

##### `close`
> public void close() throws IOException

`ByteArrayInputStream`에 이 메소드는 아무런 작동도 하지 않습니다. 스트림이 닫힌 뒤에 이 메소드가 호출되더라도 `IOException`을 일으키지 않습니다.

### ByteArrayOutputStream
#### 필드
##### `buf`
> protected byte[] buf

데이터가 저장될 버퍼입니다.

##### `count`
> protected int count

버퍼에 있는 유효한 데이터의 수입니다.

#### 생성자
> public ByteArrayOutputStream()

`ByteArrayOutputStream`을 생성합니다. 기본 버퍼 크기는 32이며, 필요한 경우 이 크기는 늘어날 수 있습니다.

> public ByteArrayOutputStream(int size)

정해진 크기의 바이트를 갖는 `ByteArrayOutputStream`을 생성합니다.

#### 메소드
##### `write`
> public void write(int b)

`ByteArrayOutputStream`에 바이트를 씁니다. 매개변수가 int인 이유는 쓸 값을 ASCII 코드 값으로 받기 때문입니다.

> public void write(byte[] b, int off, int len)

배열 `b`에서 `off` 위치부터 `len` 바이트 만큼을 이 `ByteArrayOutputStream`에 씁니다.

##### `writeBytes`
> public void writeBytes(byte[] b)

이 `ByteArrayOutputStream`에 바이트 배열에 있는 모든 내용을 씁니다.

##### `writeTo`
> public void writeTo(OutputStream out) throws IOException

`ByteArrayOutputStream`의 모든 내용을 주어진 출력 스트림에 씁니다. 이는 out.write(buf, 0, count)`와 같습니다.

##### `reset`
> public void reset()

`count`를 0으로 초기화시켜 출력 스트림에서 현재까지 축적된 출력 내용을 전부 버립니다.

##### `toByteArray`
> public byte[] toByteArray()

새로 할당된 바이트 배열을 만듭니다. 출력 스트림의 현재 크기가 이 배열의 크기가 되고 버퍼의 유효한 내용이 여기로 복사됩니다.

##### `size`
> public int size()

버퍼의 현재 크기를 반환합니다.

##### `toString`
> public String toString()

시스템에서 정해진 디코딩 방식으로 버퍼의 내용을 문자열로 바꿉니다. 새 문자열의 길이는 이 디코딩 방식에 정해져 있으므로 버퍼의 크기와 동일하지 않을 수 있습니다.

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
{: .block-warning }

> public String toString(String charsetName) throws UnsupportedEncodingException

> ##### 버전
> 자바 1.1 버전부터 지원합니다.
